# オリエンテーリング関連サービス 全体設計書

## サービス全体像

オリエンテーリング競技に関わる地図作成・大会運営・結果管理を一貫してサポートする6つのサービス。

---

## サービス一覧

| # | サービス名 | 対象ユーザー | 形態 | オフライン | 目的 |
|---|-----------|------------|------|----------|------|
| 1 | ベース地図生成 | 主催者 | Webアプリ | 不要 | 省力化 |
| 2 | 下見支援 | 主催者 | スマホ（PWA→アプリ） | 必須 | 省力化 |
| 3 | 競技地図作成 | 主催者 | Webアプリ | 不要 | 省力化 |
| 4 | CP通過チェック＆トラッキング | 参加者・主催者 | スマホアプリ | 必須 | 省力化・提供価値 |
| 5 | 結果集計 | 主催者 | Webアプリ | 不要 | 省力化 |
| 6 | 結果閲覧・ルート分析 | 参加者 | Webアプリ | 不要 | 提供価値 |
| 7 | 大会管理・エントリー | 主催者・参加者 | Webアプリ | 不要 | 省力化・提供価値 |

---

## 実施順とデータの流れ

```
サービス1: ベース地図生成（Web）
  └── 出力: PDF・GeoJSON（フェーズ1）
           GeoTIFF・MBTiles（フェーズ2以降）
        ↓
        【ファイル転送】
        フェーズ1: 手動転送（AirDrop・メール等）
        フェーズ2: アカウント連携で自動同期
        ↓
サービス2: 下見支援（スマホ・PWA→将来アプリ化）
  └── 入力: サービス1のGeoJSON（CP候補・メモ・縮尺・範囲）
  └── 出力: CP確定座標・作図データ・メモ・写真（GeoJSON）
        ↓
サービス3: 競技地図作成（Web）
  └── 入力: サービス1のGeoJSON・サービス2のGeoJSON
  └── 出力: 完成競技地図（PDF・配布用）
        ↓
サービス4: CP通過チェック＆トラッキング（スマホアプリ・オフライン対応）
  └── 入力: サービス3の競技地図
  └── 出力: CP通過記録・位置情報・トラッキングログ
        ↓
サービス5: 結果集計（Web・主催者向け）
  └── 入力: サービス4のデータ（自動取込 or 手動入力）
  └── 出力: 集計結果データ
        ↓
サービス6: 結果閲覧（Web・参加者向け）
  └── 入力: サービス5の集計結果
  └── 出力: 参加者向けリザルト表示
```

## 各サービスの詳細

### サービス1: ベース地図生成

**概要**
国土地理院の高精度DEMデータを使い、オリエンテーリング用ベース地図をWebブラウザ上で生成・ダウンロードできるサービス。

**主な機能**
```
□ 地図上でエリアを範囲選択
□ 等高線間隔をユーザーが指定
□ 建物・道路・河川レイヤーの表示
□ 磁北線の自動描画（pygeomag で偏差を自動計算）
□ 縮尺チェック（IOF標準縮尺 1:4000〜1:15000）
□ PDF（A3・コンビニ印刷対応）ダウンロード
□ GeoTIFF（レイヤー別）ダウンロード
□ MBTiles（サービス2向け）ダウンロード
```

**出力仕様**
```
フェーズ1:
  PDF:     A3・300dpi・60MB以下（コンビニ印刷対応）
  GeoJSON: CP候補・メモ・縮尺・範囲情報（サービス2連携用）

フェーズ2（将来）:
  GeoTIFF: レイヤー別（contour / buildings / roads / rivers / merged）
           座標系: JGD2011（EPSG:6668）・解像度: 600dpi / 300dpi
  MBTiles: サービス2のオフライン地図用
```

**ファイル転送方法**
```
フェーズ1（暫定）: 手動転送
  GeoJSONをダウンロード → AirDrop・メール・LINEでスマホへ転送

フェーズ2（将来）: アカウント連携
  サーバーに保存 → サービス2で自動同期
```

**開発状況**: 開発中（フェーズ1・Webアプリ）

### サービス2: 下見支援

**概要**
オリエンテーリング地図作成者が現地下見を効率的に行うためのスマホアプリ。電波なし環境での利用を前提とする。

**ファイル授受の方針**
```
フェーズ1（暫定）: 手動インポート
  サービス1でGeoJSONをダウンロード
  → スマホに転送（AirDrop・メール・LINEなど）
  → サービス2でファイル選択して読み込む

フェーズ2（将来）: アカウント連携
  サービス1でデータをサーバーに保存
  → サービス2で同じアカウントでログイン → 自動同期

※ UIは「インポート」ボタンを残しつつ
  「アカウントから読み込む」ボタンを追加する形で
  フェーズ1・2両対応できる設計にする
```

**主な機能**

事前準備（電波あり）:
```
□ サービス1のGeoJSONを読み込む
  → 以下が自動引き継ぎされる:
    ├── CP候補の座標・種類・用途・順番・ポイント
    ├── CP個別メモ
    ├── 全体メモ
    ├── 縮尺・サイズ・向き
    └── 印刷範囲（バウンディングボックス）
□ オフライン保存範囲を調整して地図をキャッシュ
  ├── 印刷範囲を初期表示
  ├── 四隅・辺をドラッグで範囲を広げられる（縮小不可）
  ├── 保存サイズ目安をリアルタイム表示
  └── PWAキャッシュ機能・全ズームレベルを保存
```

現地作業（オフライン）:
```
【地図画面】
  表示オプション（ON/OFF切り替え）:
    □ CP候補を表示
    □ 設置CPを表示
    □ 印刷範囲を表示
    □ 調査メモを表示
    □ 現在地を表示
      └── ONの場合: 常時取得 or 取得ボタン押したときのみ

  CP候補（グレー表示）:
    タップ → CP候補詳細（事前メモ閲覧）
    「現在地にCP設置」→ CP編集画面が開く（候補データ引き継ぎ）

  設置CP（赤表示）:
    タップ → CP編集画面が開く

  CP設置ボタン:
    押すと現在地のGPS座標・時刻を記録
    → CP編集画面が自動で開く

  調査メモ入力モード:
    モード切替で調査メモ入力モードへ
    ポイント: タップで配置
    ライン:   タップで点を追加 → 「完了」で確定
    エリア:   タップで点を追加 → 最初の点をタップで閉じる

【CP編集画面】
  取得情報（変更不可）:
    └── 取得緯度経度・取得時刻

  CP情報（編集可）:
    └── CP緯度経度（初期値: 取得緯度経度）
    └── 位置調整ボタン（地図上でドラッグ・矢印ボタンで微調整）
    └── 設置箇所の説明（尾根・谷など）
    └── 写真（撮影 or ファイル選択）
    └── メモ（自分用）

【調査メモ入力画面】
  カテゴリ（固定＋自由入力）:
    ポイント: 岩・崖・通行止め・水場・その他
    ライン:   トレイル・フェンス・崖（線状）・その他
    エリア:   立入禁止区域・藪・その他
  メモ（テキスト）
  写真（撮影 or ファイル選択）
```

サービス3への出力:
```
ZIPファイルとしてダウンロード:
  survey.geojson
    ├── CP候補（cp_candidate）
    │     座標・種類・用途・順番・ポイント・事前メモ
    ├── 設置CP（cp）
    │     取得座標・取得時刻・CP座標・説明・メモ・写真・引き継ぎ元候補
    └── 調査メモ（survey_memo）
          オブジェクト種別・カテゴリ・メモ・写真・座標/ライン/エリア

  photos/
    └── 写真ファイル一式
```

**技術**
```
フェーズ1:
  React（PWA）
  MapLibre GL JS（地図表示）
  国土地理院タイル（PWAキャッシュ・Service Worker）
  Geolocation API（GPS）
  IndexedDB（データ永続化）
  JSZip（ZIP出力）
  Camera API / input[type=file]（写真）

フェーズ2:
  React Native または Capacitor（アプリ化）
  MapLibre React Native
  MBTiles（オフライン地図）
  SQLite（端末内データ保存）
```

**実装前決定事項**

データの永続化:
```
保存方法: IndexedDB（ブラウザ内）
自動保存: 操作のたびに自動保存
手動保存: 保存ボタンを押したときも保存
→ ブラウザを閉じても復元できる
```

GPS取得失敗時の対応:
```
① エラーメッセージを表示
   「GPS信号を取得できませんでした」

② 3つの選択肢を提示:
   ［再試行］        → GPSを再取得
   ［緯度経度を入力］ → テンキーで直接入力
   ［地図上で指定］   → 地図の中心に十字カーソルを表示
                       地図をスクロールして位置を合わせる
                       緯度経度をリアルタイム表示
                       「この位置に設置」で確定
```

Undo機能:
```
対象: CP設置・調査メモ追加・削除・編集
操作:
  スマホ: Undoボタン（画面上）
  PC:     Ctrl+Z
最大ステップ数: 50
```

**推奨開発ステップ**
```
Step1: 事前準備画面
  └── GeoJSONインポート・地図表示・オフライン保存範囲調整

Step2: 地図画面（基本）
  └── 地図表示・現在地・CP候補表示・表示オプション

Step3: CP設置
  └── CP設置ボタン・CP編集画面・IndexedDB保存
  └── GPS失敗時の3種フォールバック

Step4: 調査メモ
  └── 調査メモ入力モード・ポイント/ライン/エリア描画

Step5: Undo・自動保存
  └── 操作履歴管理・自動保存

Step6: ZIP出力
  └── GeoJSON生成・写真まとめ・ZIPダウンロード

Step7: PWA化
  └── Service Worker・オフライン対応・ホーム画面追加
```

**開発状況**: 未着手（サービス1の後に着手予定）

### サービス4: CP通過チェック＆トラッキング

**概要**
大会当日に参加者が使うスマホアプリ。CP通過チェックと位置情報トラッキングを行い、主催者がリアルタイムで全参加者の状況を把握できる。

**主な機能**

参加者側:
```
□ CP通過チェック（写真 or QR・今後決定）
□ 通過時刻・位置情報の記録
□ 定期的な位置情報記録（例：30秒ごと）
□ 電波回復時にサーバーへ自動送信
□ オフライン対応必須
```

主催者側:
```
□ 競技地図上で全参加者の位置をリアルタイム表示
□ CP通過状況の確認
□ 全参加者のトラッキングログ確認
```

**電波なし環境への対応**
```
対策A: スタート・ゴールエリアのみWi-Fi設置
対策B: 電波があるときだけ送信（部分的なトラッキング）
対策C: 完走後にまとめてアップロード
※ どの対策を採用するかは今後決定
```

**開発状況**: 未着手・要件定義中

## GeoJSONフォーマット仕様

### ファイル命名規則

```
S1出力: {エリア名}_{日付}_s1_v{バージョン}.geojson
S2出力: {エリア名}_{日付}_s2_v{バージョン}.zip

例:
  takao_20260510_s1_v1.geojson
  takao_20260510_s2_v1.zip
```

### IDルール

| type | IDプレフィックス | 例 |
|------|----------------|-----|
| cp_candidate | cpc_ | cpc_001 |
| cp | cp_ | cp_001 |
| survey_memo | sm_ | sm_001 |

### typeの一覧

| type | 説明 | 出力元 |
|------|------|--------|
| cp_candidate | CP候補 | S1→S2引き継ぎ |

| survey_memo | 調査メモ | S2 |
